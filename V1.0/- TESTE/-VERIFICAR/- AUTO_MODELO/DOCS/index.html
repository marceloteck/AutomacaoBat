Documentação — Automação Padronizada (config.txt + dados.txt)

Um “mini DSL” (linguagem simples) baseado em TXT/INI para automações via PowerShell: 1 motor único N automações config + dados variáveis filtros

Visão geral

Este sistema usa dois arquivos TXT por automação: config.txt (regras + tarefas) e dados.txt (registros de produtores/dados).

Você cria infinitas automações apenas duplicando um RUN_*.bat e trocando quais TXT ele aponta. O motor AUTOMACAO_PADRAO.ps1 é sempre o mesmo.

Fluxo:
  1. O RUN_*.bat chama o motor e passa -Config e -Dados.
  2. O motor lê o config.txt e monta a lista de TAREFAS.
  3. O motor lê o dados.txt e monta a lista de PRODUTORES (ou outros registros, se você usar).
  4. Para cada tarefa, ele executa os PASSO1..PASSOn (com variáveis e filtros).

Estrutura de pastas

AUTOMACAO\
  CENTRAL_MENU.bat                 (menu central - opcional)
  RUN_RECEBIMENTO.bat              (launcher da automação A)
  RUN_SALVAR_PEDIDOS.bat           (launcher da automação B)
  scripts\
    AUTOMACAO_PADRAO.ps1           (motor único)
    click_positions.ps1            (opcional: CLICK:)
  input\
    recebimento_config.txt
    recebimento_dados.txt
    salvar_pedidos_config.txt
    salvar_pedidos_dados.txt
Importante: mantenha nomes consistentes e claros (ex.: xxx_config.txt e xxx_dados.txt).

Como rodar (RUN_*.bat)

Você duplica esse arquivo e muda apenas os caminhos dos TXT:

@echo off
setlocal
chcp 65001 >nul
cd /d "%~dp0"

set "MOTOR=%~dp0scripts\AUTOMACAO_PADRAO.ps1"
set "CFG=%~dp0input\recebimento_config.txt"
set "DADOS=%~dp0input\recebimento_dados.txt"

powershell -NoProfile -ExecutionPolicy Bypass -STA -File "%MOTOR%" -Config "%CFG%" -Dados "%DADOS%"
exit /b %ERRORLEVEL%

O motor aceita caminhos absolutos ou relativos. O recomendado é sempre usar %~dp0 (relativo ao BAT).

config.txt — Referência completa

1) Seções suportadas

  • [CONFIG] configurações globais do motor
  • [VAR] variáveis globais (usadas em {{VAR.X}})
  • [TAREFA] define uma tarefa (pode repetir várias vezes)

2) [CONFIG] — parâmetros

ChaveTipoPadrãoDescrição
DELAY_PADRAO_MS Número 0 Delay automático após KEY, PASTE, TYPE e CLICK.
CONFIRMAR_INICIO S/N S Se S, pergunta “Iniciar agora? (S/N)” antes de rodar.
MOSTRAR_CONTAGEM S/N S Se S, faz contagem 3..1 com beep antes de iniciar (tempo para focar o ERP).

3) [VAR] — variáveis globais

Tudo que você colocar aqui vira {{VAR.SUA_CHAVE}} nas tarefas. 

[VAR]
PASTA_EXPORT=C:\Temp\Export
OBS=Rodar com Corporate em foco

4) [TAREFA] — campos

ChaveTipoPadrãoDescrição
NOME Texto (sem nome) Nome exibido no console.
ATIVO S/N S Se N, a tarefa é ignorada.
ESCOPO GLOBAL/PRODUTOR GLOBAL GLOBAL: roda 1 vez.
PRODUTOR: roda para cada registro do dados.txt (cada [PRODUTOR]).
REPETIR Número 1 Quantas vezes repetir a tarefa (útil para loop simples).
QUANDO Expressão (vazio) Filtro condicional (somente para ESCOPO=PRODUTOR). Ex.: STATUS==PENDENTE && TIPO==PF
PASSO1..PASSOn Texto - Passos executáveis (ver seção “Passos”).

Exemplo mínimo de tarefa

[TAREFA]
NOME=COLAR INSTRUCAO
ATIVO=S
ESCOPO=PRODUTOR
QUANDO=STATUS==PENDENTE
REPETIR=1

PASSO1=SLEEP:3000
PASSO2=PASTE={{INSTRUCAO}}
PASSO3=KEY:{ENTER}
PASSO4=SLEEP:300

dados.txt — Referência completa

O dados.txt é o arquivo com os registros. No padrão atual, o motor entende: [PRODUTOR] (pode repetir várias vezes).

Campos sugeridos

Você pode criar outros campos livremente; se existirem no TXT, você consegue usar como {{CAMPO}}.

[PRODUTOR]
NOME=JANIO DE SOUSA CUNHA
STATUS=PENDENTE
TIPO=PF
INSTRUCAO=24730
PEDIDO=26059
NOTAS=NF123;NF124
Observação: os valores são tratados como texto. Em QUANDO, a comparação é exata (ex.: PENDENTE é diferente de pendente).

Variáveis {{...}}

Você pode inserir variáveis dentro de qualquer passo que aceita texto (ex.: PASTE/TYPE/ASK/OPENPATH/SETVAR):

TipoExemploResultado
Campo do PRODUTOR {{INSTRUCAO}} Cola o valor de INSTRUCAO do produtor atual
Campo do PRODUTOR {{NOME}} Nome do produtor atual
Variável global {{VAR.PASTA_EXPORT}} Busca em [VAR] do config.txt

Exemplos

PASSO1=ASK:Iniciar tarefa para {{NOME}}? (S/N)
PASSO2=PASTE={{INSTRUCAO}}
PASSO3=OPENPATH:{{VAR.PASTA_EXPORT}}
Dica: você pode criar um texto “padrão” em [VAR] e reutilizar em muitas tarefas.

Condições (QUANDO)

O campo QUANDO controla se a tarefa roda para o produtor atual. Só faz sentido quando ESCOPO=PRODUTOR.

Operadores suportados

  • == igual
  • != diferente
  • && E (AND)
  • || OU (OR)

Exemplos prontos

QUANDO=STATUS==PENDENTE
QUANDO=STATUS==PENDENTE && TIPO==PF
QUANDO=STATUS!=OK
QUANDO=TIPO==PF || TIPO==PJ
Regra: a comparação é texto exato. Se quiser padronizar, use sempre os mesmos valores (ex.: STATUS = OK/PENDENTE/ERRO).

Passos (PASSO1..PASSOn)

Os passos são a “linguagem” que executa ações. Cada PASSO é uma linha executável.

Lista completa de comandos suportados

ComandoFormatoO que faz
SLEEP SLEEP:3000 Espera em milissegundos.
KEY KEY:{ENTER} / KEY:^a Envia tecla via SendKeys. Exemplos: {F7}, ^v, +{TAB}.
PASTE PASTE=texto Coloca o texto no Clipboard e envia Ctrl+V. Aceita {{...}}.
TYPE TYPE=texto Digita literal via SendKeys (pode ser sensível a caracteres especiais). Aceita {{...}}.
ASK ASK:pergunta Pergunta S/N. Se N, aborta a tarefa atual (erro controlado). Aceita {{...}}.
CLICK CLICK:NOME Executa Invoke-ClickPos -Name no click_positions.ps1 (se existir).
OPEN_DESKTOP OPEN_DESKTOP Abre %USERPROFILE%\Downloads\DESKTOP.
OPENPATH OPENPATH:C:\Pasta Abre um caminho com o app padrão (normalmente o Explorer). Aceita {{...}}.
SETVAR SETVAR:CHAVE=VALOR Cria/atualiza variável global em runtime (vira {{VAR.CHAVE}}). Aceita {{...}}.

SendKeys — Cheatsheet rápido

{ENTER}      Enter
{TAB}        Tab
+{TAB}       Shift+Tab
^{A} / ^a    Ctrl+A
^v           Ctrl+V
^c           Ctrl+C
{F1}..{F12}  Teclas de função
{UP} {DOWN}  Setas
{LEFT}{RIGHT} Setas
Exemplos:
PASSO1=KEY:{F7}
PASSO2=SLEEP:500
PASSO3=PASTE={{INSTRUCAO}}
PASSO4=KEY:{ENTER}
PASSO5=KEY:{ENTER}
Dica importante: prefira PASTE= para textos longos e números; o TYPE= pode falhar com caracteres especiais.

Exemplos prontos (copiar e usar)

Exemplo 1 — F7 + colar instrução + Enter Enter

[TAREFA]
NOME=F7 + COLAR INSTRUCAO + ENTER ENTER
ATIVO=S
ESCOPO=PRODUTOR
QUANDO=STATUS==PENDENTE
REPETIR=1

PASSO1=ASK:Deixar ERP em foco para {{NOME}}? (S/N)
PASSO2=SLEEP:2000
PASSO3=KEY:{F7}
PASSO4=SLEEP:500
PASSO5=PASTE={{INSTRUCAO}}
PASSO6=SLEEP:300
PASSO7=KEY:{ENTER}
PASSO8=KEY:{ENTER}
PASSO9=SLEEP:800

Exemplo 2 — Rodar só PF e colar pedido

[TAREFA]
NOME=COLAR PEDIDO (SO PF)
ATIVO=S
ESCOPO=PRODUTOR
QUANDO=STATUS==PENDENTE && TIPO==PF
REPETIR=1

PASSO1=KEY:^a
PASSO2=PASTE={{PEDIDO}}
PASSO3=KEY:{ENTER}
PASSO4=SLEEP:300

Exemplo 3 — Abrir pasta Desktop no final da automação

[TAREFA]
NOME=ABRIR DESKTOP NO FINAL
ATIVO=S
ESCOPO=GLOBAL
REPETIR=1

PASSO1=OPEN_DESKTOP

Exemplo 4 — Usar variáveis globais (VAR)

[VAR]
PASTA_EXPORT=C:\Temp\Export

[TAREFA]
NOME=ABRIR PASTA EXPORT
ATIVO=S
ESCOPO=GLOBAL
REPETIR=1

PASSO1=OPENPATH:{{VAR.PASTA_EXPORT}}

Exemplo 5 — SETVAR em runtime (guardar valor para usar depois)

[TAREFA]
NOME=GUARDAR PRODUTOR ATUAL EM VAR
ATIVO=S
ESCOPO=PRODUTOR
QUANDO=STATUS==PENDENTE
REPETIR=1

PASSO1=SETVAR:ULTIMO_NOME={{NOME}}
PASSO2=ASK:Ultimo produtor foi {{VAR.ULTIMO_NOME}}. Continuar? (S/N)
Observação: SETVAR cria uma variável global que permanece enquanto o motor está rodando (até terminar a execução).

Boas práticas

  • Padronize valores: STATUS = OK / PENDENTE / ERRO (evita filtros quebrando).
  • Use PASTE para textos e números; use TYPE só quando necessário.
  • Adicione SLEEP após ações que abrem telas/menus (F7, F4, etc.).
  • Comece com ASK no início da tarefa crítica: evita rodar no lugar errado.
  • Delays curtos (200–500ms) normalmente são suficientes; telas pesadas pedem 800–1500ms.
  • Separe por tarefas: uma tarefa por “bloco lógico” facilita manutenção.

Erros comuns e como corrigir

1) “PASSO inválido”
Causa: escreveu um comando que o motor não reconhece.
Correção: use somente os formatos listados em Passos.
2) Variável vazia (ex.: {{INSTRUCAO}} não cola nada)
Causa: o campo não existe no produtor atual ou está vazio no dados.txt.
Correção: confira a chave (INSTRUCAO) e o valor no bloco [PRODUTOR].
3) O ERP não estava em foco
Sintoma: teclas e colagens vão para o lugar errado.
Correção: mantenha MOSTRAR_CONTAGEM=S e use ASK antes do bloco.
4) “CLICK usado, mas click_positions.ps1 não fornece Invoke-ClickPos”
Causa: você usou CLICK: sem ter um click_positions.ps1 compatível.
Correção: crie/ajuste scripts\click_positions.ps1 com a função Invoke-ClickPos.
5) Delay insuficiente
Sintoma: o sistema “come” teclas ou cola antes do campo abrir.
Correção: aumente SLEEP após ações de tela ou aumente DELAY_PADRAO_MS.
